# Ejercicio 3:

## Enunciado

Este proyecto contiene una aplicación web para la gestión de tareas.

Complete las funcionalidades de esta aplicación.

Si necesita añadir alguna explicación de cómo ha implementado un criterio de
evaluación, inclúyalas al final de este documento.

Nota: Dispone de mucho tiempo, organícese y vaya poco a poco. No dude en consultar
ejemplos de clase y documentación en internet.

## Criterios de evaluación

* La aplicación compila y ejecuta. (Requisito Indispensable)
  * OK | NO

* Uso de convenciones de código.                                    1    punto
      * Uso del paquete distinto al por defecto.
      * Clases, métodos y variables bien nombrados.
      * Código bien formateado.

* Funcionalidad completa de la aplicación.                          5    puntos
      * La aplicación muestra la lista de tareas                   (1    punto )
      * La aplicación permite añadir una tarea                     (1    punto )
      * La aplicación permite editar el texto de una tarea         (1    punto )
      * La aplicación permite eliminar una tarea                   (1    punto )
      * La aplicación permite marcar como completada una tarea     (0.5  puntos)
      * La aplicación permite marcar como pendiente  una tarea     (0.5  puntos)

* Implementación mejorada.                                          1    punto
      * Reutilización de archivos JSP                              (0.5  puntos)
      * Gestión de errores                                         (0.5  puntos)

Extra:

* Mejoras creativas. (3 puntos)
Justifique aquí su trabajo extra para poder acceder a esta puntuación.
Estas mejoras se puntuarán teniendo en cuenta:
 * Cantidad de trabajo necesaria.
 * Dificultad de la tarea realizada.
 * Calidad de la explicación realizada.
